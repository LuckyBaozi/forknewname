# 微信起名小程序（精简版）软件需求

## 一、项目概述
基于**纯广告变现**，取消支付功能，主打轻量化、流程极简。用户填写起名相关信息，观看完整视频广告后，获取姓名及含义解析，全程操作简单、上手无门槛。

**运行环境**：微信小程序（iOS/Android）、配套后端、微信流量主广告能力

## 二、核心业务流程（精简版）
首页 → 填写信息表单 → 提交 → 观看广告 → 展示起名结果（名字+解析）

## 三、用户角色
1. 普通用户：填写信息、观看广告、查看/收藏/分享名字
2. 后台管理员：维护字库、起名规则、广告配置、数据统计

## 四、前端功能&页面设计
### 页面1：首页
1. 页面元素
    - 小程序名称、简短标语
    - 核心按钮：**开始起名**（唯一主入口）
    - 底部小字免责提示：结果仅供娱乐参考
2. 交互：点击按钮直接跳转信息填写页

### 页面2：信息填写页（核心表单）
#### 1. 录入字段（精简，区分必填/选填）
| 字段 | 录入方式 | 必填 | 规则说明 |
| ---- | ---- | ---- | ---- |
| 姓氏 | 文本输入 | ✅ | 限1-2个汉字，过滤符号、数字 |
| 性别 | 单选 | ✅ | 男孩 / 女孩 / 中性 |
| 生肖 | 下拉选择 | ✅ | 十二生肖列表 |
| 出生年月日 | 日期选择器 | ✅ | 不可选择未来日期 |
| 出生时间 | 时间选择器 | ✅ | 支持选择「时辰未知」 |
| 地域 | 省市区选择 | ⭕选填 | 参考方言读音，可不填 |
| 名字字数 | 单选 | ✅ | 两字名 / 三字名 |
| 名字风格 | 多选标签 | ✅ | 可选：文雅、顺口、大气、温婉、古风，至少选1项 |

#### 2. 表单规则
- 所有**必填项未完成**时，「提交起名」按钮置灰并提示补充信息
- 前端实时校验姓氏格式、日期合法性
- 新增**重置**按钮：一键清空所有填写内容

#### 3. 交互
填写完毕 → 点击「提交起名」→ 校验通过 → 直接拉起**视频广告**

### 页面3：广告页（微信官方流量主视频广告）
1. 规则：必须**完整播放、正常关闭广告**才算解锁成功
2. 广告频控：**不限制每日观看次数**，用户可无限次看广告解锁结果（最大化广告收益，起名场景本身不具备刷量动机，无需频控）
3. 异常处理：
   - 中途关闭广告：提示「广告未看完，请重新提交」，返回表单页
   - 广告加载失败：提示「网络异常，广告加载失败，请稍后重试」
4. 广告播放完成 → 自动跳转**结果页**

### 页面4：起名结果页
1. 页面布局&内容
    - 顶部：**重新起名**、**返回首页**按钮
    - 主体：展示 5~8 个推荐姓名（数量后台可配置）
    - 每个姓名独立卡片，内容包含：姓名、读音、单字释义、整体寓意、风格/生肖适配说明
2. 基础功能
    - 单条名字**收藏**：本地缓存记录，可反复查看
    - 分享：支持分享小程序卡片至微信好友/群聊
3. 逻辑：同一组信息，重复查看结果**无需再次看广告**（前端将表单参数序列化 hash，命中本地缓存直接展示）

### 页面5：我的收藏（辅助页面）
1. 入口：结果页/首页增设「我的收藏」入口
2. 功能：展示所有已收藏名字及完整解析，支持单条删除、清空全部
3. 存储：微信本地缓存，换设备不保留（精简开发，不上云同步）

## 五、后端核心能力（精简）
### 1. 名字库管理
- 维护汉字库：标注五行、生肖宜用字/忌用字、生僻字黑名单
- 按风格、性别、字数分类管理名字及对应解析文案
- 名字库初始数据来源：人工编撰 + 公开起名资料整理，确保内容合规

### 2. 起名匹配规则（名字库检索模式）
**核心原则：不做实时算法生成，从预建名字库中检索匹配。**

匹配流程：
1. 根据用户选择的**性别 + 字数 + 风格标签**从名字库中筛选候选名字
2. 按**生肖宜忌**过滤：排除含忌用字的名字，含宜用字的加权靠前
3. 过滤**生僻字黑名单**中的名字
4. 结合**出生年月日 + 地域**做辅助排序（同分下地域适配优先）
5. 返回 Top N 条结果（N 后台可配，默认 8）

**名字库数据模型：**
```
名字 | 性别 | 字数 | 风格标签 | 生肖适配 | 读音 | 单字释义 | 整体寓意 | 地域适配(可选)
```

### 3. 广告配置
- 绑定微信流量主广告位 ID，后台可开关广告、切换广告类型

### 4. 基础数据统计
- 统计访问量（UV/PV）、表单提交数、广告播放完成数、名字结果查看数

### 5. 接口能力
- 接收前端表单数据 → 按检索规则匹配名字库 → 返回结果列表

## 六、后台管理端（轻量化方案）
采用**单文件 Web 页面 + JSON 文件存储**方案，不搭建独立后台系统。

### 实现方式
- 名字库以 **JSON 文件** 形式维护，存储在后端项目目录中
- 管理操作通过**命令行脚本**或**本地 Web 管理页**完成（仅管理员本地访问）
- 数据统计在**小程序云开发控制台**或后端日志中查看，不单独开发统计后台

### 管理能力
1. 名字库 CRUD：增删改查名字及对应解析文案
2. 配置调整：修改返回名字数量、广告位 ID、风格标签枚举
3. 生僻字黑名单维护

### 管理员鉴权
- 管理页面设置简单密钥（环境变量配置），无需用户系统

## 七、技术选型（轻量化优先）

| 层次 | 方案 | 理由 |
|------|------|------|
| 小程序前端 | 微信原生框架 | 无额外依赖，5个页面原生开发最快 |
| 后端 | **微信云开发（CloudBase）** | 免服务器运维、自带数据库和存储、与小程序天然集成、免费额度够用 |
| 数据库 | 云开发数据库（文档型） | 名字库以文档形式存储，无需建表，直接 JSON 导入 |
| 缓存 | 云开发数据库本地缓存 + 小程序端 Storage | 高频名字结果自动缓存，减少数据库读取 |
| 广告 | 微信流量主「激励视频广告」 | 原生能力，接入简单 |

> **备选方案**：如果不想绑定微信云开发，可用 **Express.js + SQLite**（一个文件跑起来），部署在单台轻量云服务器上即可。

## 九、全局规则&异常处理
### 1. 通用规范
- 全页面布局适配手机各尺寸，加载流畅
- 全程无额外弹窗广告，仅解锁环节播放激励视频广告
- 所有页面标注免责声明：起名结果仅作娱乐参考

### 2. 异常场景处理

| 场景 | 处理方式 |
|------|----------|
| 网络中断 | 统一提示「网络异常，请检查网络后重试」 |
| 广告中途关闭 | 提示「广告未看完，请重新提交」，返回表单页 |
| 广告加载失败 | 提示「网络异常，广告加载失败，请稍后重试」 |
| 无匹配名字 | 提示「暂无合适名字，请调整风格/生肖后重试」 |
| 接口请求超时 | 提示重试按钮，支持点击重新请求 |
| 表单校验不通过 | 对应字段标红提示，提交按钮置灰 |

## 十、整体页面跳转链路
```
首页 → 信息填写页 → 提交表单 → 播放激励视频广告 → 结果页
                                                  ├→ 重新起名 → 填写页（保留原有内容）
                                                  ├→ 返回首页
                                                  ├→ 收藏名字 → 我的收藏
                                                  └→ 分享 → 微信分享面板
```
